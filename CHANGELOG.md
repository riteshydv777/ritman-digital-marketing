## How to Update

### Minor Updates (bug fixes, small features)
1. Update code
2. Bump patch version (1.0.1)
3. Commit with message: `fix: description`
4. Redeploy to Vercel

### Feature Releases (new sections, components)
1. Update code
2. Bump minor version (1.1.0)
3. Commit with message: `feat: description`
4. Update CHANGELOG
5. Tag release on GitHub
6. Redeploy to Vercel

### Major Releases (breaking changes, redesign)
1. Plan thoroughly
2. Create feature branch
3. Implement changes
4. Test extensively
5. Update all documentation
6. Bump major version (2.0.0)
7. Create pull request
8. Merge and deploy
